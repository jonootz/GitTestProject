Branches

  • Multiple branches can be entirely independent of each other.
  • Uses:
    • Try out an ideas, merge back in
      • Flare has no UI to Merge Branches, but you can via GitBash (command line)
    • Experiment in, then delete branch
    • Traditionally used to test New Features or hot fixes (Think of Flare dot-releases)
    • Have one that goes to production, another that you merge work into for testing, and several smaller ones for day to day work

Things to check in Flare:

  • Check to make sure files and changes are retained in correct branch when switching
  • Make sure before creating a new branch commit, push and pull, your master needs to be up to date.
  • With Git you can switch between branches without having to commit. With Flare we force you to.

Things to check in Remote:

Pushing a branch within Flare should push only changes in that branch to remote.

 

Things to check in database via Git:

 

git checkout -b <branch_name>:

Creates new branches

 

git checkout <branch_name>:

Switches between branches

 

git branch -D [name_of_your_new_branch]:

Deletes branches

 

git log:

Checks commits related to branch. Will include commits from master before it was branched.

 

Check Windows Explorer to see if branches have different files
