Committing

Committing changes or additions to the repository (Local Database).

Things to check:

  1. In Pending Changes:
    • Committed files should not display
  2. In File List files should show status as "Committed"

Things to check in Remote:

Remote only shows Commits whereas the true working repository can be checked with GitBash and Flare (limited).

 

Things to check in database via Git:

 

git status:

Only uncommitted files should list under "Changes not staged for commit" or "Untracked files"

OR

If a Commit All is done, then "On branch master. Nothing to commit, working directory clean" should display

 

git log:

New log entry should display.

Example:

commit 30adae31ab035736839729c9e9c973d3b990053d
Author: jharkins <jh@jh01.com>
Date:   Mon Dec 21 12:24:35 2015 -0800
	Comment goes here
