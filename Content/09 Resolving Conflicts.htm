Resolving Conflicts

Demo / Test:

  1. Dirty a paragraph in User A's project > Commit > Push
  2. Dirty the same paragraph in User B's project > Commit > Push
    1. When second user forces a conflict, error states: "Specified remote repository has been updated, perform a pull prior to pushing to remote."
    2. User B should Pull
      1. Resolve Conflicts dialog appears

 

Things to check in Flare:

      1. Pull

    2. If user discards local:

      1. Pull
      2. Resolve conflict keeping server version

    3. If user discards external:

Things to check in database via Git:

 

git status:

During the MERGE:

Your branch and 'origin/master' have diverged, and have 1 and 1 different commit each, respectively

 

After the resolve:

Your branch is ahead of 'origin/master' by 2 commits.

 

After pushing:

"On branch master. Your branch is up-to-date with 'origin/master'. Nothing to commit, working directory clean" should display

 

git log:

After User B resolves issues you should have 3 commits in log: User A's change, User B's change, and the merge commit
